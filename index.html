/ 01 / CORE CAPABILITIES

memory that works like memory

not a vector store. not a key-value cache. real bi-temporal memory with decay, graph traversal, and zero external dependencies.

01.0CORE

bi-temporal storage

every fact has valid-from and transaction time. track what was true when, not just what's true now. facts have history.

$ cairn remember "uses vim"
stored: editor:vim (t=now)
02.0CORE

graph traversal

walk the memory graph. find related facts by connection, not just similarity. context that chains across sessions.

$ cairn graph "editor"
vim -> dark-theme -> linux
03.0DX

zero config

no api keys, no embeddings, no cloud account. one binary, one sqlite file. ship in minutes, not sprints.

$ cairn init
created memory.db
/ 02 / USAGE

no sdk to learn, no client to install, no dashboard to configure. cairn runs from your terminal.

/ 03 / HOW IT WORKS

three steps, zero magic

no embeddings pipeline. no vector index. no inference server. just sqlite and regex.

01

Remember

cairn parses text with regex and keyword extraction. facts get stored with confidence and timestamps.

$ cairn remember "likes vim"
stored: editor:vim (1.0)
02

Recall

query by keyword or graph traversal. cairn returns facts ranked by confidence and recency, not similarity.

$ cairn recall "editor"
vim (0.92, 2h ago)
03

Forget

confidence decays automatically. facts below threshold get pruned. your memory stays clean without manual cleanup.

$ cairn forget --stale
pruned 4 facts
/ 04 / ARCHITECTURE

a vector db stores chunks.
cairn stores facts.

vector databases return the same text chunks every time. cairn tracks how facts evolve, contradict, and expire. one sqlite file. zero llm calls.
