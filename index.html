how it works

three steps, zero magic

no embeddings pipeline. no vector index. no inference server. just sqlite and regex.

1

remember

cairn parses text with regex and keyword extraction. facts get stored with confidence and timestamps.

$ cairn remember "user likes vim"
stored: editor:vim (1.0)
2

recall

query by keyword or graph traversal. cairn returns facts ranked by confidence and recency, not similarity.

$ cairn recall "editor"
vim (0.92, 2h ago)
3

forget

confidence decays automatically. facts below threshold get pruned. your memory stays clean without manual cleanup.

$ cairn forget --stale
pruned 4 facts
architecture

a vector db stores chunks.
cairn stores facts.

vector databases return the same text chunks every time. cairn tracks how facts evolve, contradict, and expire.
