what makes it different

not a knowledge graph. not a vector store. wayfinding.

agents leave cairns to mark what they learned and where they've been. like hikers stacking stones on a trail. the category isn't memory. it's finding your way back.

01 — temporal

nothing gets deleted

facts have validity windows. new facts close old ones. query what was true on any date. history isn't erased, it's superseded.

02 — extraction

zero LLM calls

patterns catch "i use X", "i prefer X", "i work at X". the agent calls remember() as an MCP tool. no extra inference, no extra cost.

03 — forgetting

decay, not deletion

confidence decays over time. low-confidence facts get tombstoned. user-stated facts (confidence >= 0.8) never auto-forget. auditable.

04 — sync

federated, not centralized

vector clocks. CRDT-style merge. conflict resolution by confidence, then recency, then device id. no server required.

05 — query

no query language

keywords find entry points. graph traversal walks the subgraph. ranking by confidence and recency. returns json. no cypher, no sparql.

06 — storage

one file

a single sqlite database. no neo4j. no postgres. no cloud. copy the file, you've copied the memory. that's the whole backup strategy.

install
# clone and build $ git clone https://github.com/valtors/cairn $ cd cairn && cargo build --release # store what your agent learns $ cairn remember --subject tamish --predicate uses_editor --object vim remembered: 71838f5f-18d7-46f1... # ask it later $ cairn recall "tamish uses_editor" vim (confidence: 1.0, recorded: 2026-07-22) # extract from conversation, zero LLM $ cairn extract "i use rust and i live in bangalore" extracted: 2 facts, 2 stored # run as MCP server $ cairn serve --stdio
