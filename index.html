01 features

six primitives. one file.

focused tools for storing, recalling, and forgetting what agents learn. no cloud, no server, no llm dependency.

01

bi-temporal storage

every fact has valid-from and transaction time. track what was true when, not just what's true now.

02

decay-based forgetting

stale facts fade automatically. confidence decays over time. agents forget what no longer matters.

03

zero-llm extraction

keyword extraction and regex patterns. no api calls, no tokens spent. facts are parsed, not generated.

04

graph traversal query

follow edges between facts. find related context by walking the memory graph, not just keyword matching.

05

vector clocks

conflict detection across sessions. concurrent writes are tracked, not silently overwritten.

06

one sqlite file

the entire memory layer is one sqlite database. copy it, back it up, inspect it with any sqlite tool.

02 vs legacy

a vector db stores chunks. cairn stores facts.

vector databases return the same text chunks every time. cairn tracks how facts evolve, contradict, and expire.

vector db

stores chunks. returns chunks. each session starts from zero.

  • requires a running server
  • cosine similarity is the only tool
  • no concept of time or staleness
  • chunks can't contradict or evolve
  • costs scale with embedding calls

cairn

facts that evolve. knowledge that merges, contradicts, and gets forgotten across sessions.

  • one sqlite file, no server needed
  • graph traversal + keyword search
  • bi-temporal: tracks when facts were true
  • conflict detection via vector clocks
  • zero llm calls for extraction
